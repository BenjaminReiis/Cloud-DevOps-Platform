Serviços do Google Cloud envolvidos

Cada serviço cumpre um papel específico na cadeia de deploy.

ServiçoPapel no pipeline
Artifact RegistryArmazena as imagens Docker versionadas do backend e do frontend.
Cloud Run (Backend)Executa a API Node.js como serviço serverless, escalando sob demanda.
Cloud Run (Frontend)Serve o bundle React de forma independente do backend.
Cloud SQLBanco de dados relacional gerenciado, provisionado via Terraform.
Cloud Operations SuiteCentraliza logs e métricas dos serviços em produção.
04 · automação

O workflow, de ponta a ponta

GitHub Actions cuida do build e push das imagens; Terraform cuida da infraestrutura.

.github/workflows/deploy.yml
name: Deploy to GCP
on:
  push:
    branches: [main]

jobs:
  build-and-push:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build backend image
        run: docker build -t $REGISTRY/backend:$GITHUB_SHA ./backend
      - name: Build frontend image
        run: docker build -t $REGISTRY/frontend:$GITHUB_SHA ./frontend
      - name: Push to Artifact Registry
        run: |
          docker push $REGISTRY/backend:$GITHUB_SHA
          docker push $REGISTRY/frontend:$GITHUB_SHA

  terraform-deploy:
    needs: build-and-push
    runs-on: ubuntu-latest
    steps:
      - uses: hashicorp/setup-terraform@v3
      - name: Apply infrastructure
        run: |
          terraform init
          terraform apply -auto-approve \
            -var "image_tag=$GITHUB_SHA"
infra/cloud_run.tf
resource "google_cloud_run_v2_service" "backend" {
  name     = "cloudpay-backend"
  location = "us-central1"

  template {
    containers {
      image = "${var.registry}/backend:${var.image_tag}"
      env {
        name  = "DATABASE_URL"
        value = google_sql_database_instance.main.connection_name
      }
    }
  }
}
